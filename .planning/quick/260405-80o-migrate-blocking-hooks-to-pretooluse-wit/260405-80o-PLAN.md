  <name>Task 1: Add dual-mode output to the 3 blocking hook scripts</name>
  <files>hooks/completion-audit.sh, hooks/dev-cycle-check.sh, hooks/ci-status-check.sh</files>
  <action>
Each of the 3 scripts currently outputs PostToolUse format: `{"decision":"block","reason":"...","hookSpecificOutput":{"message":"..."}}`.

Add hook_event_name detection near the top of each script (after reading stdin JSON into `$input`):

```bash
hook_event=$(printf '%s' "$input" | jq -r '.hook_event_name // "PostToolUse"')
```

Then create a helper function in each script that outputs the correct format based on the event:

```bash
emit_block() {
  local reason="$1"
  local json_reason
  json_reason=$(printf '%s' "$reason" | jq -Rs '.')
  if [[ "$hook_event" == "PreToolUse" ]]; then
    printf '{"hookSpecificOutput":{"hookEventName":"PreToolUse","permissionDecision":"deny","permissionDecisionReason":%s}}' "$json_reason"
  else
    printf '{"decision":"block","reason":%s,"hookSpecificOutput":{"message":%s}}' "$json_reason" "$json_reason"
  fi
}
```

Replace ALL `printf '{"decision":"block",...}'` calls with `emit_block "$msg"` (or equivalent variable). Keep non-blocking informational messages (exit 0, status messages) unchanged — those only matter in PostToolUse context and are harmless in PreToolUse.

For each script specifically:

**completion-audit.sh**: Has multiple block outputs (lines 168-169, 173-174, 189-190). Replace each with `emit_block "$msg"`.

**dev-cycle-check.sh**: Has block outputs at lines 187-188 (Stage A) and 209-210 (Stage B), plus the third-party boundary block at lines 43-44. Replace all with `emit_block`.

**ci-status-check.sh**: Has one block output at lines 55-56. Replace with `emit_block`.

IMPORTANT: The `emit_block` function and `hook_event` variable must be defined BEFORE any code path that calls them. Place them right after `input=$(cat)` and the initial jq extractions.
