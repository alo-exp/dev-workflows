## CONS-02 ‚Äî Required-skills schema drift

### Current state

- `hooks/lib/required-skills.sh` defines `DEFAULT_REQUIRED` (15 skills, includes `review-loop-pass-1/2`) and `DEVOPS_DEFAULT_REQUIRED`.
- `templates/silver-bullet.config.json.default` `skills.required_deploy` has 11 skills (no `review-loop-pass-1/2`, no `silver-blast-radius` devops variant, no `devops-quality-gates`).
- Divergence known (backlog 999.25).

### Fix

Make `templates/silver-bullet.config.json.default` the single source of truth. Add devops arm as an additional key so both variants have a canonical home.

- Add `skills.required_deploy_devops` array to the default config (alongside `required_deploy`).
- Refactor `hooks/lib/required-skills.sh` into a reader-shim that parses the default config at source-time with `jq` (fallback: the repo root `templates/silver-bullet.config.json.default`; if absent, the in-cache copy shipped with the plugin).
- Verify downstream hooks that source the lib see unchanged `DEFAULT_REQUIRED`/`DEVOPS_DEFAULT_REQUIRED` strings.

Align content: extend the default config to match the 15-skill enforced list (add `review-loop-pass-1`, `review-loop-pass-2`; add `silver-blast-radius` + `devops-quality-gates` in the devops arm).

### Deliverable

`tests/hooks/test-required-skills-consistency.sh`:

1. Parse `DEFAULT_REQUIRED` and `DEVOPS_DEFAULT_REQUIRED` from the lib.
2. Parse `skills.required_deploy` and `skills.required_deploy_devops` from the default config.
3. Diff as sorted sets; fail on any divergence.
