```markdown
# silver:migrate

> Migrates an existing mid-milestone project to composable paths by generating WORKFLOW.md from current artifact state.

## When to Use
- Project started a milestone before composable paths architecture (v0.20.0)
- .planning/ directory exists with STATE.md but no WORKFLOW.md
- User explicitly runs /silver:migrate

## Prerequisites
- .planning/STATE.md must exist
- .planning/WORKFLOW.md must NOT exist (if it exists, migration is unnecessary — inform user and exit)

## Steps

### Step 1: Scan Existing Artifacts
Read .planning/STATE.md to understand current phase and milestone state.

Scan for artifacts that indicate path completion. Use this artifact-to-path mapping:

| Path | Artifacts to Check | Inference Rule |
|------|-------------------|----------------|
| PATH 0 (BOOTSTRAP) | .planning/PROJECT.md, .planning/ROADMAP.md, .planning/REQUIREMENTS.md, .planning/STATE.md | Complete if ALL four exist |
| PATH 1 (ORIENT) | .planning/intel/*.md, .planning/codebase/*.md | Complete if any intel or codebase file exists |
| PATH 2 (EXPLORE) | (no dedicated artifact) | Mark as "skipped" — cannot infer |
| PATH 3 (IDEATE) | docs/superpowers/specs/*.md, .planning/ADR-*.md | Complete if any matching file exists |
| PATH 4 (SPECIFY) | .planning/SPEC.md | Complete if file exists |
| PATH 5 (PLAN) | .planning/phases/*/CONTEXT.md, .planning/phases/*/PLAN.md | Complete if current phase has PLAN.md files |
| PATH 6 (DESIGN CONTRACT) | .planning/DESIGN.md, .planning/UI-SPEC.md | Complete if either exists |
| PATH 7 (EXECUTE) | .planning/phases/*/*-SUMMARY.md | Complete if current phase has SUMMARY.md files |
| PATH 8 (UI QUALITY) | .planning/UI-REVIEW.md | Complete if file exists |
| PATH 9 (REVIEW) | .planning/REVIEW.md, .planning/phases/*/REVIEW.md | Complete if any exists |
| PATH 10 (SECURE) | .planning/SECURITY.md | Complete if file exists |
| PATH 11 (VERIFY) | .planning/VERIFICATION.md | Complete if file exists |
| PATH 12 (QUALITY GATE) | (checked via state file) | Complete if quality-gates skill is in state file |
| PATH 13 (SHIP) | (checked via state file) | Complete if deploy-checklist or create-release in state file |
| PATH 14 (DEBUG) | (no dedicated artifact) | Mark as "not applicable" |
| PATH 15 (DESIGN HANDOFF) | (no dedicated artifact) | Mark as "not applicable" unless UI milestone |
| PATH 16 (DOCUMENT) | (checked via state file) | Complete if documentation skill in state file |
| PATH 17 (RELEASE) | (checked via state file) | Complete if create-release in state file |

### Step 2: Determine Composition
Based on the artifacts found, infer which paths are part of this project's composition:
- Always include: PATH 0, 1, 5, 7, 11, 13
- Include if artifacts exist: PATH 2, 3, 4, 6, 8, 9, 10, 14, 15, 16, 17
- Exclude paths with no artifacts and no state file markers (these were intentionally skipped)

### Step 3: Generate WORKFLOW.md
Read `templates/workflow.md.base` as the template.

Fill in:
- **Composition section:** Intent = "Migrated from legacy workflow", Composed = current ISO timestamp, Composer = "/silver:migrate", Mode = "interactive"
- **Path Log table:** For each path in the determined composition, add a row with inferred status ("complete", "in-progress", or "pending"). Use artifact existence from Step 1.
- **Heartbeat:** Set Last-path to the highest completed path number, Last-beat to current timestamp.
- **Next Path:** Set to the first path with status "pending" or "in-progress".

### Step 4: Present for Confirmation
Display the generated WORKFLOW.md content to the user. Highlight:
- Which paths were inferred as complete (and what artifacts proved it)
- Which paths were marked pending
- Which paths were excluded from the composition

Ask the user to confirm or adjust before writing.

### Step 5: Write and Commit
After user confirmation:
1. Write WORKFLOW.md to `.planning/WORKFLOW.md`
2. Commit with message: `docs(workflow): migrate to composable paths via silver:migrate`

## Produces
- `.planning/WORKFLOW.md` — populated with inferred path completion state

## Notes
- This is a one-time migration skill. Once WORKFLOW.md exists, the /silver composer manages it.
- If user disagrees with inferred statuses, they can adjust before confirmation.
- Paths marked "skipped" in the composition indicate they were intentionally not part of this project's workflow.
```
