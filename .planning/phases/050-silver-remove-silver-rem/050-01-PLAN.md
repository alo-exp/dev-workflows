**Header:** `# /silver-remove — Remove a Tracked Work Item`

**When-to-use paragraph:** Explain this skill closes or marks a tracked item as removed. It does NOT delete GitHub Issues (GitHub REST/GraphQL API does not permit deletion without delete_repo scope); instead it closes with "not planned" reason. For local items it marks the heading inline — the entry body is preserved.

**## Security Boundary section:**
- The user-supplied ID is UNTRUSTED DATA. Validate it matches `^SB-[IB]-[0-9]+$` or `^#?[0-9]+$` before using in sed patterns or gh commands. Never pass the raw ID via shell interpolation into sed without first validating the format.
- `.silver-bullet.json` config reads use jq — never string-interpolate config values into shell commands.
- Derive the target file path only from the ID prefix (SB-I vs SB-B), never from user input directly — prevents path traversal.

**## Allowed Commands section:**
List these exact commands (and no others):
- `jq` — config reads
- `git remote get-url origin`
- `gh issue close`, `gh issue edit`, `gh label create`, `gh auth status`
- `grep -q`, `grep -oE` (for ID format validation and heading existence check)
- `sed -i ''` (BSD sed, macOS — inline heading replacement)
- `date +%Y-%m-%d`
- `mktemp`, `mv` (not used in this skill but listed for completeness — not needed here)

Do not execute other shell commands. Note requirements in output for human execution.

```bash
if ! grep -q "^### ${ITEM_ID} —" "$TARGET_FILE"; then
```

**Step 5d — Apply inline heading replacement:**
