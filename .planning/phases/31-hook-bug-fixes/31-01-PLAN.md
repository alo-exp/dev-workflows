**hooks/uat-gate.sh -- Fix line 47 (Check 2: FAIL detection)**

Replace the current single-grep FAIL check on lines 47-48:
```bash
if grep -qE '\| FAIL \|' "$UAT"; then
  fail_count=$(grep -cE '\| FAIL \|' "$UAT" || true)
```

With a two-pipe approach that excludes header rows:
```bash
if grep -E '\| FAIL \|' "$UAT" | grep -qvE '\|\s*(#|Total|PASS|NOT.?RUN|Status|Result)\s*\|'; then
  fail_count=$(grep -E '\| FAIL \|' "$UAT" | grep -cvE '\|\s*(#|Total|PASS|NOT.?RUN|Status|Result)\s*\|' || true)
```

Logic: First grep finds all lines with `| FAIL |`. Second grep (`-v`) excludes lines that also contain header-indicator strings (`#`, `Total`, `PASS`, `NOT-RUN`, `Status`, `Result`). A header row like `| # | Criterion | PASS | FAIL | NOT-RUN | Total |` matches the exclusion pattern (contains `#`, `PASS`, `Total`, etc.) and is filtered out. A data row like `| 2 | Logout | FAIL |` does NOT match the exclusion and triggers the block.

The `fail_count` line gets the same two-pipe treatment for an accurate count of data-row failures.

**hooks/dev-cycle-check.sh -- Fix lines 139-141 (state tamper detection for Bash commands)**

Replace the current two-grep check on lines 139-141:
```bash
    if [[ "$is_whitelisted_append" == false ]] && \
       printf '%s' "$command_str" | grep -qE '\.claude/[^/]+/(state|branch|trivial|mode)' && \
       printf '%s' "$command_str" | grep -qE '(>>|\s>[^>&=]|\btee\b)'; then
```

With a single combined grep that requires the write operator to PRECEDE the state path:
```bash
    if [[ "$is_whitelisted_append" == false ]] && \
       printf '%s' "$command_str" | grep -qE '(>>|\s>[^>&=]|\btee\b)[^<]*\.claude/[^/]+/(state|branch|trivial|mode)'; then
```

Key change: The `[^<]*` bridge between the write operator and the state path ensures:
1. `echo x >> ~/.claude/.silver-bullet/state` STILL matches (write op before state path, no `<` between them)
2. `cat > /tmp/foo << 'EOF'\n~/.claude/.silver-bullet/state\nEOF` does NOT match (the `<` in `<<` stops the `[^<]*` from bridging to the state path in the heredoc body)

Update the comment on lines 135-137 to reflect the combined approach:
```bash
    # Bash write to .silver-bullet/state, /branch, or /trivial -> block
    # Single combined pattern: write operator must precede state path with no heredoc
    # delimiter (<) between them. This prevents false-positives from heredoc body content.
```
